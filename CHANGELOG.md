### Added
- **Chat provider system COMPLETE (Milestone 11, 8/8)** ‚Äî generic multi-provider backend shipped on `feature/chat-provider-system`, not yet merged. The chat system now speaks two protocols: Claude Code CLI (`runClaudeStreaming`, session-id-based history) and any OpenAI-compatible `/v1/chat/completions` API (`runOpenAIStreaming`, stateless, history-per-request). Provider routing happens via a new `server/providers/router.ts::routeToProvider()` generator that looks up provider config from `db.providers`, resolves credentials (api-key direct, oauth via auto-refreshing `getValidToken`, or none), and delegates to the right backend ‚Äî yielding `StreamChunk` in the same shape regardless of source so the SSE fan-out is unchanged. New modules: `server/providers/openai-adapter.ts` (SSE parse + fetch streaming + AbortSignal + 60s inactivity timeout), `server/providers/oauth.ts` (OAuth 2.0 auth code flow with CSRF state, 5-minute expiry buffer, disconnect preserves config), `server/providers/model-discovery.ts` (Ollama `/api/tags`, OpenAI-compat `/v1/models`, Claude known set; 60s TTL cache that deliberately does NOT cache failures so a down provider recovers immediately). `db.providers` seeded with built-ins (`claude-code`, `ollama` ‚Äî Ollama respects `OLLAMA_URL` env). CRUD at `/api/providers` with `sk-...<last4>` api-key masking, `oauthTokens`/`clientSecret` stripped from wire, `builtin: true` blocks deletion. OAuth endpoints (`/auth`, `/auth/callback`, `/disconnect`, `/status`) use a browser-popup flow (`window.open(authUrl, '_blank')` + 3s post-open status refetch) ‚Äî no BroadcastChannel, no polling loop. New Settings ‚Üí Chat tab (`client/src/components/settings/provider-manager.tsx`) with add/edit dialog, capability checklist (7 flags), OAuth connect/disconnect buttons, availability indicators driven by the same `useProviderModels` hook the composer uses (so the React Query cache is shared). Composer wiring finished: `chat-settings-store.ts` now holds a `providers` slice fetched from `/api/providers` via `loadProviders()`; the hardcoded `BUILTIN_PROVIDERS`/`MODEL_CATALOGS` static imports from M10's `builtin-providers.ts` are gone from runtime client code (file reduced to test-fixture-only, 4 test suites still reference it ‚Äî full deletion deferred).
